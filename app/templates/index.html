Step 1

Our team searched for a dataset with quality python code. We knew that better data for training the AI would result in a better product. Finally, we found a dataset on Kaggle, containing more than enough code to train our model.

Step 2

For our project, our team had found a dataset full of Python code. However, all of the code was stored in nested folders. Using recursion in Python, we retrieved the data from the folders and concatenated it into a text file. We also tried doing this in the Linux Terminal. Next, we had to split the data into smaller parts because it was too large of a dataset. We used random in Python to get random samples of the data. Lastly, we found that our data had to be cleaned. Our team members worked together and removed the docstrings and comments from the code, using Python.

Step 3

The next step was to train the AI model. Since we used Aitextgen, a library in Python that is used to generate text, we only needed to fine tune our model using the training data that we gathered before. We set the model to train for 12,000 steps. Our learning rate was set to 5e-3, and the AI generated text every 1000 steps.

Step 4

We tested out our AI Code Generator manually by entering different prompts in Google Colab. Based on the results, we processed the data again, and added more data. We found a better quality dataset, though it had less in quantity, and cleaned it, adding it to the original dataset. Then, we trained the AI, and re-evaluated. Finally, we used autopep8 as a python linter, which helped make the code look better.

Step 5

We created the website using Bootstrap and revised it in html code. Collaborating on Replit, we made the front-end of our product.

Step 6

Using Flask, the website and the AI parts of our product were merged together. This is where we are now.
